# 为什么选择Mac？

前言

很早之前个人也是觉得苹果的配置又低，好像也没什么性价比，还卖的的这么贵；直到我身边的人开始用苹果，我开始拿着他们的设备开始做对比， 然后时不时会去体验一下苹果的产品（就是南京东路最大那一家苹果店），苹果的产品配置和同类产品配置比起来根本没什么可比性，但是苹果的产品体验很上瘾； 自己的设备最先开始是接触iPad开始的，然后一发不可收拾（成为了一名果粉）...

「消费者并不知道自己需要什么，直到我们拿出自己的产品，他们就发现，这是我要的东西。」

# 我的旧友

我的第一台笔记本是小黑 [ThinkPad E系列]，最让人怀恋的还是那让人欲罢不能的小红点/斜眼笑，陪伴了我五年的青葱青春岁月，也是时候让你退休了。 咳咳，回到正题～

# 个人的一些体验与感受

# 外观方面

怀着激动的心情拆开了刚到的最新款 MacBook Pro (16-inch, 2019) ，一开始最让人不熟悉的其实是键盘，看着好像多了很多奇怪的符号一样， 其实对比下来只是把 win 键和 alt 键调换了位置，就好比 ThinkPad 把 fn 和 ctrl 调换了位置一样， 相信在座的各位肯定也有保存的时候按成 fn + s 的/斜眼笑，还是需要一点时间适应的；
厚度方面比我的小黑基本上薄了一半，也没我的小黑那么重，然鹅有些人觉得很重很大不方便，不适合出差；他们可能喜欢13寸那个亚子，当然我喜欢屏幕大。
Touch Bar是第一次使用，不喜欢每个软件之间变化可以设置一直显示 F1 - F12

# 一些体验

4k的分辨率，环绕的音效，感觉给人带来了一场视听盛宴，用了一段时间网上说有爆音的问题，暂时还没有遇到，没用过上万的win本，可能都差不多把，这里不说太多。 不过触摸板极度舒适，我已经离不开了，但是键盘经常敲代码的话还是没有外接键盘舒适。

# 系统方面

系统方面倒是很快完成了转换，因为 Mac OS 也是基于 Unix 的系统，装软件也是非常的傻瓜式， 双击打开拖一下就好了，也有 Homebrew 包管理器； Linux 也可以去官网下载安装包双击安装，或者通过包管理器在命令行安装，一般选择后者，因为逼格高/滑稽， 个人非常喜欢苹果系统的UI，之前读书的时候将 Deepin 、 Ubuntu 做为主力开发系统； 本人也是一名终端党，喜欢命令行，因为敲命令可以让你理解一些操作的流程原理，而操作界面点几下你根本不知道他做了哪些事，不过各有各的好处， 通过自己判断选择适合自己的方法，像多人开发通过代码管理系统更新大量代码的时候如果怕有冲突，我会选择IDEA，因为有图形化的合并，点选自己想要的地方就行了， 而命令行只是在文件中做标记，还得手动去除冲突的代码。

# 一些感受

以前用 Linux 的时候最喜欢的主题、图标啥的也都是仿 Mac 的，一开始使用 Deepin 是因为开箱即用，基本上就是当 Windows 用； 因为他们自己封装的 wine 安装qq是所有linux里面最稳定的，后面转 ubuntu 也是用的deepin的包；让人苦恼的是linux的桌面环境不稳定，至少我东西开多一点就会假死， 也有可能是配置低的原因，不像windows会无响应然后恢复，而linux只能强制重启，如果东西没保存就凉凉了。Mac 桌面比较稳定，默认特效叫 “神奇效果”，和 KDE 桌面环境的特效一摸一样，也不知道是谁抄谁的。这里用过的一些 Linux 桌面环境： Unity Gnome KDE Xfce

# 软件方面

Linux 开源，没钱赚，很多软件厂商不提供 Linux 版本，很多都是 Github 上的大佬自己封装的，要么就是封装的网页版，要么通过wine装win上的软件； 而对 Mac 的软件支持稍微多一些，这也是我选择它的原因之一，当然不会有 Windows 那么多；Mac 软件支持差不多处于 Linux 和 Windows 之间，至于以后这么多端 是否是鸡肋还是会转型合并，就交给时间来验证把。

# 文件系统

基于 Unix 的系统没有和 Windows 一样的盘符概念，对于开发人员来说路径其实比 Windows 的更方便；例如：
E:\backup\project.log 很多编程语言中会不认识 \ 字符，可能会把它识别成转译字符， 通常我们在 Windows 引用路径的时候需要用 \\ 来表示 \：E:\\backup\\project.log。

/backup/project.log 则还是 /backup/project.log
对以后部署项目有一定帮助，大部分项目都部署在 Linux 系统上。
